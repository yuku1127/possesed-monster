----------操作説明------------

キーボードの操作割り当ては以下の通りです。  
D:攻撃  A:ヒョーイ/ヒョーイ解除（遠距離攻撃）  W:インベントリを開く  Space:ジャンプ
  
PCの音量を最小にしておくことを推奨します。
ゲーム画面の大きさは開始する前のブラウザのウィンドウサイズに設定されるため、あらかじめフルスクリーンにしておくことを推奨します。
またゲーム終了はブラウザを閉じる、または再読み込みで終了します

本作品に使用されている一部の素材は、以下の提供元より使用許諾を得て利用しています

----------仕様解説------------

---main.jsの役割---

全てのスクリプトの処理を最終的にはこのmain.jsで毎フレーム実行する。各クラス共通の変数,関数をまとめて定義し、各クラスに一つだけ必要なインスタンスはmain.js実行時に一度だけ生成(行番号81~92)、それ以外のクラス（行番号72~）はmap.js(次の項目参照)のタイルを識別して複製する。
メインループ関数（行番号230~）の中でupdate関数(入力された内容からデータ更新)とdraw関数（更新された値を描画に反映）をGAME_FPSの周期で繰り返し呼び出す
プレイヤーの入力はイベントハンドラで管理し各クラス共通の変数値を変更する

--—map.jsの役割—--

16*16ピクセルのタイル約７０種類をTiledというマップエディタを使ってステージに配置し、それを各タイルに番号をつけたfieldData配列に変換[(map.js行番号１〜より)
プレイヤーの座標から仮想キャンバス範囲内のタイルを全て描画（map.js行番号197のdraw関数）、特定のタイルからはモンスターやギミックのあるオブジェクトを生成

----------アピールポイント------------

---敵、ギミックの生成---

map.jsの説明でも書きましたが、モンスタークラス(monster.js)などのインスタンスを特定のマップ上のタイルの番号から呼び出すことで、負荷を抑え、また少しでも視覚的にオブジェクトを管理できるように工夫しました。

---プレイヤーの処理---

プレイヤーは敵であるモンスターに電波を浴びせ乗り移って攻略を進めていきます（紹介動画参照）。それを実現するために発射した電波に触れたモンスターの情報をプレイヤーに上書きし(atack.jsの行番号80~)、player.jsから実行するモンスターの挙動を管理しました。(行番号238)

---ボスの挙動---

攻撃モーションは３種類あり、近接攻撃、遠距離攻撃、全体攻撃があります（monster.js行番号460~525）。プレイヤーは事前に乗り移ったモンスターの攻撃でもボスを倒せますが、ボスの全体攻撃で生成されるモンスター（トゲねずみ）を利用しても倒せるようにしました（紹介動画参照）。具体的にはmonster.jsのcheckfloor関数で落石を落とす処理を呼び出し（monster.js行番号54）,bossFunc1()の中でトゲネズミを生成するattak.jsのインスタンスを生成します（monster.js行番号538）。
